## 追问与补充

### 需要补充的信息

1. **SQL 使用场景**：请问你用 SQL 具体做过什么？是查询数据还是做分析？频率如何？
2. **PRD 输出频率**：你每月大约输出多少份 PRD？独立还是协同？
3. **跨团队协作**：你说的"对接客户和研发"，你的具体角色是什么？协调了多少人？
4. **数据分析深度**：你说"参与数据分析"，是查看后台报表还是做分析结论？
5. **个人贡献边界**：订单系统中，你个人具体负责了什么模块？

### 用户补充

- SQL 每月约 2-3 次，主要用于取数验证产品数据
- PRD 独立输出，每月 1-2 份
- 协调过 5 名研发和 1 名 UI
- 主要做后台数据查看，未形成过独立分析报告
- 订单系统中独立负责"订单查询"模块

## 风险提示

**高风险（影响初筛）**：
- 3 年经验 vs JD 要求 5 年，可能被机筛淘汰

**中风险（影响面试）**：
- SQL 使用描述需更具体，避免"偶尔取数"印象

**低风险（可准备）**：
- 会员改版参与经历，可准备个人亮点
