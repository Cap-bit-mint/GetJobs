# 常见失败模式

本文档列出 GetJobs Skill 在执行过程中可能出现的失败模式，以及识别信号和修正方式。

## JD 相关失败模式

### 1. 编造 JD 内容

**识别信号**：
- JD 内容过于通用/模糊
- 与用户提供的信息矛盾
- 缺少具体数字或细节

**修正方式**：
```
⚠️ 无法自动获取 JD 内容。
请提供：
1. JD 文本粘贴
2. JD 来源链接
3. JD 文件上传
```

---

### 2. 使用过期职位

**识别信号**：
- 职位已下架但仍显示
- 发布时间超过 30 天
- 招聘平台显示「已结束」

**修正方式**：
```
⚠️ 检测到该职位可能已下架或过期。

建议：
1. 确认职位是否仍在招聘
2. 提供最新的 JD 内容
3. 搜索是否有新职位开放
```

---

### 3. JD 抓取不完整

**识别信号**：
- 缺少薪资、福利等信息
- 职责/要求部分缺失
- 无法提取完整内容

**修正方式**：
```
⚠️ JD 内容不完整，缺少以下信息：
- [缺失字段]

当前 JD 仍可用于基础分析。
如需完整评估，请补充上述信息。
```

---

## 匹配分析失败模式

### 4. 过度匹配（虚假高匹配分）

**识别信号**：
- 匹配分远高于预期
- 把「弱覆盖」写为「已覆盖」
- 忽略明显的技能缺口

**修正方式**：
```
重新评估匹配分：
- SQL：应判定为「弱覆盖」，非「已覆盖」
- 匹配分修正：78 → 65

当前评估过于乐观，请核实：
1. 简历中 SQL 的实际使用场景
2. 项目中是否有数据分析深度
```

---

### 5. 忽略硬性条件

**识别信号**：
- 硬性条件不满足但未标注
- 未提示「机筛淘汰风险」
- 匹配分未反映硬性条件缺失

**修正方式**：
```
⚠️ 硬性条件缺失：
- JD 要求：5年+经验
- 简历显示：3年经验

风险：可能被机筛淘汰

建议：
1. 如实告知用户风险
2. 不应为了「好看」忽略此风险
3. 可建议尝试内推渠道绕过机筛
```

---

### 6. 贡献强度失真

**识别信号**：
- 简历写「参与」，分析写「负责」
- 无数据支撑的成果归因
- 团队成果全部归给个人

**修正方式**：
```
⚠️ 贡献强度评估错误：
- 原评估：「主导」跨团队项目
- 实际描述：「参与」跨团队项目

动词强度需修正：
- 主导 → 参与/协助

只有满足以下条件才能使用「主导」：
1. 对目标、方案、节奏承担主要责任
2. 能说明个人决策如何影响结果
```

---

## 追踪管理失败模式

### 7. 记录丢失

**识别信号**：
- 查询时记录不存在
- 状态更新后记录消失
- 批量操作后部分丢失

**修正方式**：
```
⚠️ 记录不存在：[ID]

可能原因：
1. ID 输入错误
2. 记录已被删除
3. 文件损坏

建议：
1. 查询所有记录确认
2. 检查 ~/.getjobs/applications.json 文件
3. 如有备份，尝试恢复
```

---

### 8. 状态流转错误

**识别信号**：
- 状态从 rejected 变为 pending
- 状态跳过中间步骤
- 状态历史不连续

**修正方式**：
```
⚠️ 状态流转无效：rejected → pending

有效流转：
pending → viewed/interview/rejected/withdrawn
interview → offer/rejected/withdrawn

不可逆向流转（如 rejected → pending）
如需重新投递，请创建新记录。
```

---

### 9. 跟进日期计算错误

**识别信号**：
- 跟进日期超过合理范围
- 未考虑周末/节假日
- 状态更新后未重置跟进日期

**修正方式**：
```
⚠️ 跟进日期计算错误

原始：2026-06-21（周日）
修正：2026-06-22（周一）

建议跟进日期计算规则：
- 投递后：+7天（工作日）
- 查看后：+3-5天（工作日）
- 面试后：+1-2天
```

---

## 求职信失败模式

### 10. 模板化求职信

**识别信号**：
- 与简历 Skill 输出高度重复
- 包含「感谢您百忙之中审阅」等套话
- 多个公司使用相同段落

**修正方式**：
```
⚠️ 求职信过于模板化

问题：
1. 开头套话应删除
2. 应加入公司具体信息
3. 每个公司应有差异化表述

建议：
- 删除通用套话
- 加入：「我了解到贵司在[具体业务]...」
- 针对 JD 强调点定制段落
```

---

### 11. 夸大或编造内容

**识别信号**：
- 使用简历中不存在的技能
- 声称的数据无法验证
- 把「参与」写成「主导」

**修正方式**：
```
⚠️ 求职信内容与简历不符

问题：
- 求职信：「精通SQL和Python」
- 简历：「偶尔用SQL取数，无Python经验」

修正：
改为「具备SQL基础，能够进行数据取数分析」

必须保证求职信与简历内容一致。
```

---

### 12. 长度不当

**识别信号**：
- 超过 500 字（中文）
- 少于 100 字（精简版）
- 信息密度过低

**修正方式**：
```
⚠️ 求职信长度建议

中文标准版：300-500 字
中文精简版：100-200 字（Boss直聘）
英文版：250-400 words

当前字数：[XXX] 字，建议精简。
```

---

## 简历联动失败模式

### 13. 未经授权调用

**识别信号**：
- 未询问用户直接调用简历 Skill
- 自动生成简历未确认
- 多个 JD 同时调用

**修正方式**：
```
⚠️ 调用简历 Skill 需要用户授权

在调用前必须：
1. 确认用户请求
2. 确认目标 JD
3. 确认用户简历已提供

请回复「是」确认调用。
```

---

### 14. JD 未确认时生成简历

**识别信号**：
- JD 内容为空或不完整
- 用户未确认 JD 来源
- 简历版本与 JD 不匹配

**修正方式**：
```
⚠️ 无法生成定制简历

原因：
- JD 内容不完整
- JD 来源未确认

请先：
1. 提供完整 JD
2. 或确认已抓取的 JD 内容
3. 再请求生成定制简历
```

---

### 15. 生成的简历未保存

**识别信号**：
- 用户询问简历文件路径
- 输出后未更新追踪记录
- 简历内容与追踪记录不匹配

**修正方式**：
```
✅ 简历已生成，正在保存...

保存位置：~/.getjobs/output/resumes/[公司]-[岗位]-v1.md

追踪记录已更新：
- 简历版本：v1
- 简历路径：[路径]
```

---

## 浏览器自动化失败模式

### 16. 页面加载超时

**识别信号**：
- 等待元素超过设定时间
- 页面内容为空
- 网络请求失败

**修正方式**：
```
⚠️ 页面加载超时

可能原因：
1. 网络问题
2. 页面反爬机制
3. 目标页面不存在

建议：
1. 检查 URL 是否正确
2. 尝试手动访问
3. 请求用户提供 JD
```

---

### 17. 需要登录

**识别信号**：
- 页面显示登录表单
- 内容被限制访问
- 出现验证码

**修正方式**：
```
⚠️ 该页面需要登录

平台限制：
- 部分招聘平台内容需要登录才能查看

建议：
1. 提供您已登录的 JD 链接
2. 直接粘贴 JD 文本
3. 使用官网招聘页（如有）
```

---

### 18. 反爬拦截

**识别信号**：
- 页面显示「访问频率过高」
- 出现验证码或人机验证
- IP 被封禁提示

**修正方式**：
```
⚠️ 触发反爬机制

建议：
1. 等待片刻后重试
2. 直接提供 JD 内容
3. 使用用户提供的 JD 链接
```

---

## 隐私安全失败模式

### 19. 敏感信息未脱敏

**识别信号**：
- 身份证号出现在输出中
- 完整手机号未处理
- 银行信息被记录

**修正方式**：
```
⚠️ 检测到敏感信息

必须脱敏：
- 身份证号：[已删除]
- 手机号：138****8000

是否继续处理？（确认脱敏后继续）
```

---

### 20. 凭证泄露风险

**识别信号**：
- 尝试保存登录信息
- 询问用户账号密码
- 记录平台登录凭证

**修正方式**：
```
⚠️ 不保存任何登录凭证

GetJobs 不会：
- 要求您提供任何账号密码
- 保存任何平台的登录信息
- 尝试绕过登录验证

如需访问需要登录的内容，请手动提供 JD 内容。
```

---

## 一般失败模式

### 21. 配置文件损坏

**识别信号**：
- JSON 格式错误
- 文件权限不足
- 目录不存在

**修正方式**：
```
⚠️ 配置文件错误

可能原因：
1. JSON 格式错误
2. 文件权限不足
3. 目录不存在

建议：
1. 检查 ~/.getjobs/ 目录是否存在
2. 检查 applications.json 格式
3. 如需重建，备份后删除重新创建
```

---

### 22. 备份冲突

**识别信号**：
- 多个备份版本
- 最新备份覆盖问题
- 恢复后数据丢失

**修正方式**：
```
✅ 自动备份已创建

备份文件：~/.getjobs/applications.json.backup.[timestamp]

如需恢复：
1. 备份当前文件
2. 复制备份文件
3. 重命名为 applications.json
```
