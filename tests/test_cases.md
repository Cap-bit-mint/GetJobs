# 测试用例

## 测试 1：JD 搜索与抓取

### 输入

```
帮我搜索字节跳动的产品经理 JD。
```

### 预期行为

1. 识别搜索意图
2. 构建搜索策略（官网招聘页 > 招聘平台 > 用户提供）
3. 尝试抓取 JD（API 或浏览器）
4. 若失败，提示用户提供 JD

### 预期输出

```markdown
## JD 搜索结果

### 搜索策略
- 优先：官网招聘页 (jobs.bytedance.com)
- 次选：Boss 直聘、拉勾
- 兜底：请求用户提供

### 抓取尝试
[根据实际情况输出]

### 若抓取成功
| 字段 | 内容 |
|------|------|
| 公司 | 字节跳动 |
| 岗位 | 产品经理 |
| 薪资 | 面议（未标注）|
| 来源 | [链接] |
| 可信度 | 高/中/低 |

### JD 详情
[完整 JD 内容]

### 若抓取失败
⚠️ 无法自动抓取字节跳动的产品经理 JD。

建议操作：
1. 提供官网招聘链接
2. 直接粘贴 JD 文本
3. 提供 JD 文件
```

### 不应出现

- 编造 JD 内容
- 使用过期的职位信息
- 声称抓取成功但内容为空

## 测试 3：简历匹配度分析

### 输入

```
分析匹配度：
JD要求：3年+B端经验，SQL，PRD输出，跨团队推进
我的简历：
- 3年B端产品经验
- 负责订单系统，PRD输出
- 偶尔用SQL取数
- 参与跨团队项目
```

### 预期诊断

| JD 要求 | 权重 | 简历证据 | 匹配度 |
|---------|------|----------|--------|
| 3年+B端经验 | 高 | 3年B端产品 | ✅ 已覆盖 |
| SQL | 高 | 偶尔取数 | ⚠️ 弱覆盖 |
| PRD输出 | 高 | 负责PRD | ✅ 已覆盖 |
| 跨团队推进 | 中 | 参与 | ⚠️ 弱覆盖 |

### 预期匹配分

```
综合匹配分：72/100

硬性条件：95/100（年限满足）
技能匹配：60/100（SQL弱）
经验相关度：80/100（B端经验匹配）
成果量化：60/100（缺数据）
表达清晰度：70/100
```

### 预期追问

```
为准确评估，建议补充：

1. SQL 使用场景：查询频率、具体分析内容？
2. 跨团队项目：你的具体角色、推进了什么？
3. 有无量化的项目成果？
```

### 不应出现

- 把"偶尔取数"写成"熟练SQL"
- 把"参与"写成"主导"
- 给无法验证的技能打满分
