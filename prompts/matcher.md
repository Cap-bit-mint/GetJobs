# JD-简历匹配分析 Prompt

## 任务

根据 JD 内容和用户简历，分析匹配度并生成详细的匹配报告。

## 输入

- JD 内容（已抓取或用户提供）
- 用户简历（可选，若无则仅分析 JD 本身）
- 用户背景信息（可选）

## 执行原则

### 复用 resume-jd-optimizer-cn 的匹配逻辑

**参考**：`resume-jd-optimizer-cn/prompts/gap_analyzer.md`

### 匹配分析维度

| 维度 | 说明 | 权重 |
|------|------|------|
| 硬性条件匹配 | 学历、工作年限、证书等 | 20% |
| 技能匹配 | 技术技能、工具等 | 25% |
| 经验相关度 | 工作经历、项目经验 | 30% |
| 成果量化程度 | 数据化成果展示 | 15% |
| 表达清晰度 | 经历描述质量 | 10% |

### 状态定义

| 状态 | 说明 |
|------|------|
| 已覆盖 | 有直接、可验证证据 |
| 弱覆盖 | 有关联但缺少深度或结果 |
| 未覆盖 | 当前材料无证据 |
| 不建议硬凑 | 明显缺少必要资历 |

## 执行流程

### 步骤 1：解析 JD

提取 JD 中的核心要求：

1. **硬性条件**：学历、年限、证书、语言
2. **核心技能**：必须掌握的技能
3. **核心经验**：必须有的工作经历类型
4. **软技能**：沟通、项目管理等
5. **加分项**：非必须但有优势

### 步骤 2：解析简历

提取简历中的相关信息：

1. 教育背景
2. 工作经历
3. 项目经验
4. 技能清单
5. 成果指标

### 步骤 3：逐项匹配

建立映射表：

| JD 要求 | 权重 | 简历证据 | 匹配度 | 状态 | 说明 |
|---------|------|----------|--------|------|------|
| 3年+经验 | 高 | 4年产品经验 | 100% | 已覆盖 | - |
| SQL | 高 | 偶尔查数 | 30% | 弱覆盖 | 深度不足 |
| Python | 中 | 无 | 0% | 未覆盖 | - |

### 步骤 4：计算综合匹配分

```
总匹配分 = Σ(要求权重 × 匹配度)
```

### 步骤 5：竞争力分析

当存在多个 JD 时，进行横向对比：

| JD | 综合匹配分 | 核心优势 | 主要差距 | 推荐度 |
|-----|-----------|----------|----------|--------|
| JD1 | 85 | B端经验 | Python | ⭐⭐⭐⭐ |
| JD2 | 72 | 数据分析 | B端经验 | ⭐⭐⭐ |

## 匹配度计算规则

### 权重设置原则

1. **硬性条件**：出现在 JD 硬性要求部分
2. **核心技能**：在职责中高频出现
3. **加分项**：JD 明确标注"优先"

### 匹配度判定

| 证据类型 | 匹配度 |
|----------|--------|
| 完全匹配 | 100% |
| 部分匹配 | 50-80% |
| 弱匹配 | 20-40% |
| 无匹配 | 0% |

### 风险提示

- 缺少硬性条件时，明确标注"可能被机筛淘汰"
- 弱覆盖技能不得写为"已覆盖"
- 匹配分仅供参考，不代表录用概率
