# 投递追踪 Prompt

## 任务

帮助用户管理求职投递记录，包括创建、更新、查询投递状态。

## 输入

- 操作类型（创建/更新/查询/删除）
- 投递信息（公司、岗位、匹配分等）
- 状态变更信息

## 追踪文件

### 默认路径

```
~/.getjobs/applications.json
```

## 状态流转

### 状态定义

| 状态 | 说明 | 颜色标识 |
|------|------|----------|
| pending | 已投递，待查看 | 🟡 黄色 |
| viewed | HR 已查看 | 🔵 蓝色 |
| interview | 进入面试 | 🟢 绿色 |
| offer | 收到 offer | ⭐ 金色 |
| rejected | 被拒绝 | 🔴 红色 |
| withdrawn | 自己撤回 | ⚪ 灰色 |

### 状态流转规则

```
pending
  ├── viewed (HR 查看)
  │     ├── interview (进入面试)
  │     │     ├── offer (收到 offer)
  │     │     └── rejected (面试后被拒)
  ├── rejected (直接被拒)
  └── withdrawn (主动撤回)

interview
  ├── offer
  ├── rejected
  └── withdrawn
```

## 操作命令

### 1. 创建投递记录

**输入**：
```
添加投递记录：
公司：xxx
岗位：xxx
平台：boss
JD链接：xxx
匹配分：85
薪资范围：30-50k
地点：北京
```

**执行**：
1. 读取现有追踪文件
2. 生成新的 UUID
3. 添加新记录
4. 保存文件
5. 返回确认信息

**输出**：
```
✅ 投递记录已创建

| 字段 | 内容 |
|------|------|
| ID | abc123 |
| 公司 | xxx |
| 岗位 | xxx |
| 状态 | pending |
| 跟进日期 | 2026-06-20 |

下次跟进提醒：投递后 7 天
```

---

### 2. 更新投递状态

**输入**：
```
更新状态：
ID: abc123
新状态: viewed
备注: HR 已查看简历
```

**执行**：
1. 查找对应记录
2. 更新状态
3. 添加状态历史
4. 计算下次跟进日期
5. 保存文件

**输出**：
```
✅ 状态已更新

| 字段 | 旧值 | 新值 |
|------|------|------|
| 状态 | pending | viewed |
| 更新日期 | - | 2026-06-13 |

建议跟进日期：2026-06-17（查看后 3-5 天）
```

---

### 3. 查询投递记录

**输入**：
```
查询投递记录
条件：[all/pending/viewed/interview/rejected]
```

**输出**：
```markdown
## 投递记录查询结果

**总计**：12 条记录

### 按状态分布

| 状态 | 数量 |
|------|------|
| 待跟进 | 3 |
| 已查看 | 2 |
| 面试中 | 1 |
| 已拒绝 | 5 |
| Offer | 1 |

### 详细记录

| 公司 | 岗位 | 匹配分 | 状态 | 投递日期 | 跟进日期 |
|------|------|--------|------|----------|----------|
| A公司 | 产品经理 | 88 | 🟢 面试中 | 06-01 | - |
| B公司 | 高级PM | 85 | 🔵 已查看 | 06-05 | 06-10 |
| C公司 | PM | 72 | 🔴 已拒绝 | 05-28 | - |
```

---

### 4. 删除投递记录

**输入**：
```
删除记录：
ID: abc123
确认：是
```

**执行**：
1. 确认用户意图
2. 软删除（标记为 deleted）或硬删除
3. 更新文件
4. 返回确认

**输出**：
```
✅ 记录已删除

删除的记录：
- A公司 - 产品经理
```

---

### 5. 添加跟进备注

**输入**：
```
添加备注：
ID: abc123
备注内容：面试官反馈技术栈匹配度不错，但管理经验略有不足
```

**执行**：
1. 查找记录
2. 追加备注（保留历史）
3. 保存文件

**输出**：
```
✅ 备注已添加

记录：A公司 - 产品经理

备注历史：
1. 2026-06-13: HR 已查看简历
2. 2026-06-15: 约下周一面试
3. 2026-06-18: 面试官反馈技术栈匹配度不错，但管理经验略有不足
```

## 数据导出

### 导出格式

支持导出为：
- JSON（完整数据）
- CSV（表格数据）
- Markdown（报告格式）

### 导出命令

```
导出记录：
格式：[json/csv/md]
条件：[all/pending/interview]
路径：~/Desktop/求职记录.csv
```

## 异常处理

### 追踪文件不存在

```markdown
未找到追踪文件，将创建新文件。

默认路径：~/.getjobs/applications.json

确认创建？（是/否）
```

### 记录 ID 不存在

```markdown
未找到对应记录：abc123

请检查 ID 是否正确，或查询所有记录确认：

[查询所有记录]
```

### 状态流转无效

```markdown
状态流转无效：rejected → pending

当前状态：rejected
目标状态：pending

无法将已拒绝的投递改回待跟进状态。

如需重新投递，请创建新记录。
```
