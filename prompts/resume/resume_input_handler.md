# 简历输入处理规范

## 概述

本规范定义简历优化工作流的输入处理环节，确保只有经过用户确认的真实简历内容才能进入后续分析流程。

## 支持的简历格式

| 格式 | 处理方式 | 工具 |
|------|----------|------|
| 纯文本 | 直接使用 | - |
| PDF | pdf-parse 提取文本 | `tools/pdf_parser.md` |
| DOCX | 解析 XML 结构 | 内置解析 |
| 图片/截图 | OCR 提取文字 | tesseract.js |

## 输入验证流程

```
用户输入
    │
    ├── 文本粘贴 ──────────────────→ 直接使用
    │
    ├── PDF 文件
    │       │
    │       ▼
    │   解析 PDF
    │       │
    │       ├── 成功 → 展示摘要 → 用户确认
    │       │
    │       └── 失败 → 提示用户提供文本版本
    │
    └── 其他文件
            │
            ▼
        检查是否为简历（通过文件名/内容判断）
            │
            ├── 是 → 解析 → 展示摘要 → 用户确认
            │
            └── 否 → 请用户明确说明
```

## 解析后展示摘要模板

当解析完成时，向用户展示摘要：

```markdown
## 📄 简历内容已解析

**检测到以下内容**：

| 项目 | 内容 |
|------|------|
| 教育背景 | X 段（最后学历：xxx） |
| 工作经历 | X 段（共 X 年） |
| 项目经验 | X 个 |
| 技能特长 | X 项 |
| 时间线 | 20XX - 20XX |

**前 3 条工作经历预览**：
1. [公司名] · [岗位] · [时间]
2. [公司名] · [岗位] · [时间]
3. [公司名] · [岗位] · [时间]

---

请确认：这是您的简历内容吗？
- 如有遗漏或错误，请指出
- 如需补充其他内容（如作品链接），请提供
```

## 拒绝执行条件

以下情况必须立即停止简历优化流程：

### 情况 1：未收到简历内容

```markdown
⚠️ **未收到您的简历内容，无法进行简历优化。**

请选择以下方式提供简历：
1. 直接粘贴简历文本
2. 上传简历文件（PDF/DOCX）

我会自动解析文件内容并请您确认。
```

### 情况 2：简历内容过少

```markdown
⚠️ **收到的简历内容过少（少于 100 字），无法进行分析。**

请补充完整简历内容，包括：
- 教育背景
- 工作/实习经历
- 项目经验（如有）
- 技能特长（如有）
```

### 情况 3：简历内容无法解析

```markdown
⚠️ **无法解析您上传的简历文件。**

可能原因：
- 文件已加密或损坏
- 文件格式不支持
- 内容为扫描件（无文字层）

请选择：
1. 提供文本格式的简历
2. 提供截图
3. 直接粘贴简历内容
```

## 与 JD 处理的关联

简历和 JD 是独立的输入流：

```
简历处理 ──────────────────────────┐
    │                               │
    ▼                               ▼
解析 → 摘要 → 用户确认         解析 → 摘要 → 用户确认
    │                               │
    ▼                               ▼
进入事实底稿                    进入 JD 要求表
    │                               │
    └───────────┬───────────────────┘
                ▼
           缺口分析 + 简历重写
```

**关键原则**：
- 简历和 JD 都必须分别确认
- 两者的内容确认不能互相替代
- 只有两者都确认后才能执行完整优化流程

## 后续处理

确认简历内容后，进入 `resume_parser.md` 执行简历解析：
- 提取教育、工作、项目、技能
- 标记证据状态
- 识别表达问题
- 生成事实底稿
