# JD 搜索与抓取 Prompt

## 任务

根据用户提供的公司名、岗位名或其他条件，搜索并抓取目标职位的 JD 内容。

## 输入

- 公司名称（必填）
- 岗位名称（必填）
- 平台偏好（可选）
- 地点偏好（可选）
- 薪资范围（可选）

## 执行原则

### 数据来源优先级

1. **官网招聘页**（最高优先级）
   - 信息最准确、更新最快
   - 通常有独立的招聘官网

2. **招聘平台**（次选）
   - Boss 直聘、拉勾、猎聘等
   - 需尊重平台规则

3. **用户直接提供**（兜底）
   - 用户粘贴 JD 文本
   - 用户上传 JD 文件

### 抓取策略

```
1. 优先使用 API（若已配置）
2. 次选 browser 插件自动化抓取
3. 降级到用户手动提供 JD
```

## 执行流程

### 步骤 1：确定搜索目标

解析用户输入，确定：
- 目标公司（全称/简称）
- 目标岗位（精确名称/关键词）
- 地点偏好
- 其他筛选条件

### 步骤 2：构建搜索策略

**国内公司搜索顺序**：
1. 公司名 + "招聘官网"
2. 公司名 + "拉勾/Boss直聘"
3. 公司名 + "智联/前程无忧"

**海外公司搜索顺序**：
1. 公司名 + "careers" 或 "jobs"
2. LinkedIn 公司页
3. Indeed

### 步骤 3：执行抓取

**API 模式**（若已配置）：
```markdown
使用配置的 API 服务获取职位信息。
```

**Browser 模式**：
```markdown
使用 browser 插件执行以下操作：
1. navigate_to(search_url)
2. wait_for(page_load)
3. extract_text(jd_selector)
4. 若页面需要登录，提示用户提供 JD 内容
```

**用户提供模式**：
```markdown
无法自动抓取，请用户提供 JD 内容。
```

### 步骤 4：解析 JD 内容

提取以下字段：

| 字段 | 说明 | 提取规则 |
|---|---|---|
| company | 公司名 | 原文提取 |
| position | 岗位名 | 原文提取 |
| department | 部门 | 可能包含 |
| location | 工作地点 | 需标准化 |
| salary_range | 薪资范围 | 需标注单位 |
| job_type | 工作类型 | 全职/实习/远程 |
| experience | 经验要求 | 年限范围 |
| education | 学历要求 | 本科/硕士等 |
| requirements | 任职要求 | 分条提取 |
| responsibilities | 岗位职责 | 分条提取 |
| preferred | 加分项 | 分条提取 |
| benefits | 福利待遇 | 分条提取 |
| source_url | 来源链接 | 完整 URL |
| source_platform | 来源平台 | 公司官网/招聘平台 |
| posted_date | 发布时间 | 若可见 |
| crawled_at | 抓取时间 | 当前时间 |

## JD 质量评估

### 可信度分级

| 等级 | 标准 |
|---|---|
| 高 | 官网直连、招聘平台官方数据 |
| 中 | 第三方聚合平台、猎头发布 |
| 低 | 社交媒体、非官方渠道 |

### 质量检查项

- [ ] 公司名称与用户提供一致
- [ ] 岗位名称与用户需求匹配
- [ ] 薪资范围有效（若标注）
- [ ] 联系方式完整（若需要）
- [ ] 发布时间在合理范围内
- [ ] 无明显过时信息

## 异常处理

### 抓取失败

```markdown
无法自动抓取 [公司] 的 [岗位] JD。

可能原因：
- 页面需要登录
- 平台限制爬取
- 职位已下架

建议操作：
1. 提供官网招聘链接，我尝试抓取
2. 直接粘贴 JD 文本
3. 上传 JD 文件
```

### JD 不完整

```markdown
抓取的 JD 缺少以下信息，建议补充：
- [缺失字段]

当前 JD 仍可用于基础匹配分析。
```

### 多个相似职位

```markdown
发现 [N] 个相关职位：

| # | 岗位名 | 地点 | 来源 |
|---|---|---|---|
| 1 | xxx | xxx | [链接] |

请确认目标职位：
A. 选择具体岗位
B. 全部抓取分析
C. 提供具体职位链接
```
