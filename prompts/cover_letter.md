# 求职信生成 Prompt

## 任务

为用户的定制简历生成配套的求职信（Cover Letter），用于正式投递。

## 输入

- 目标公司
- 目标岗位
- 用户简历（已优化版本）
- 简历 Skill 的输出（若有）
- JD 摘要
- 用户背景信息

## 求职信结构

### 标准结构

```
1. 开场白（自我介绍 + 得知渠道）
2. 核心价值主张（为什么选我）
3. 相关经历佐证（1-2 个核心案例）
4. 对公司的理解（为什么选这家公司）
5. 结语（期待机会）
```

### 精简结构（用于 Boss 直聘等）

```
1. 简短自我介绍 + 核心匹配点
2. 1 个核心案例
3. 表达求职意向
```

## 写作原则

### 必须包含

1. **针对性**：每封求职信针对特定公司和岗位
2. **真实性**：只使用已确认的事实和经历
3. **简洁性**：通常 300-500 字，中文为主
4. **具体性**：用具体数据或案例支撑

### 禁止包含

1. **空话**：夸大能力、陈述而非证明
2. **模板腔**："感谢您百忙之中审阅"等套话
3. **与 JD 无关的内容**：只说 JD 需要的
4. **薪资要求**（除非 JD 明确要求）
5. **弱点**：不提自己的不足

## 执行流程

### 步骤 1：提取关键信息

从简历和 JD 中提取：

1. **公司信息**：
   - 公司名称
   - 公司业务/产品
   - 公司文化/特点（若有）

2. **岗位信息**：
   - 岗位名称
   - 核心要求（来自 JD）
   - 匹配点

3. **个人优势**：
   - 最相关的 2-3 个经历
   - 最匹配的 2-3 个技能
   - 最有说服力的 1-2 个数据

### 步骤 2：匹配分析

建立映射：

| JD 要求 | 个人经历 | 匹配度 |
|---------|----------|--------|
| 3年+ B端经验 | 4年B端产品经验 | 强匹配 |
| SQL 数据分析 | 独立搭建周报体系 | 中匹配 |

### 步骤 3：撰写求职信

按结构撰写：

```markdown
## 求职信

尊敬的 HR：

您好！

我是 [姓名]，目前从事产品工作 [X] 年，关注到贵司正在招聘 [岗位名]。

在 B 端产品领域，我有 [X] 年的经验，曾独立负责 [核心项目]，实现了 [量化成果]。

[选择最匹配的 1-2 个经历展开]

我了解到，贵司 [公司特点]，这与我的职业方向高度契合。

期待有机会与您进一步沟通。

此致
敬礼

[姓名]
[联系方式]
[日期]
```

## 输出格式

### 中文求职信（标准版）

```markdown
## 求职信

尊敬的 [公司名] HR：

您好！

我是 [姓名]，[当前职位/最近职位]，从事产品工作 [X] 年。

[核心价值主张 + 1-2 个具体案例]

[量化数据或成果]

[对公司/岗位的理解]

期待有机会进一步交流，感谢您的时间。

此致
敬礼

[姓名]
[手机号]
[邮箱]
[日期]
```

### 英文求职信

```markdown
## Cover Letter

[Date]

I am writing to express my strong interest in the [Position] role at [Company].

With [X] years of experience in [Field], I have developed expertise in [Skills]...

I am particularly drawn to [Company] because [Reason]...

Thank you for your consideration. I look forward to discussing how I can contribute to your team.

Best regards,
[Name]
[Phone]
[Email]
```

### 精简版（BOSS 直聘）

```markdown

[姓名] | [X]年产品经验

您好！我看到贵司正在招聘[岗位名]，很感兴趣。

我的核心优势：

2. [匹配点2]：[量化成果]

[一句话对公司的理解]

期待进一步沟通~
```

## 差异化版本

### 根据 JD 要求调整

| JD 强调 | 求职信重点 |
|---------|-----------|
| 经验 | 用更长篇幅描述相关经历 |
| 技能 | 强调技能掌握程度和实际应用 |
| 学历 | 提及教育背景（若有优势） |
| 团队管理 | 强调管理经验和团队成果 |

### 根据公司调整

| 公司类型 | 求职信风格 |
|----------|-----------|
| 大厂 | 专业、量化、结果导向 |
| 创业公司 | 灵活、主动性、适应能力 |
| 外企 | 英文能力、跨文化沟通 |
| 传统企业 | 稳定性、专业积累 |

## 常见问题

### Q：需要为每个 JD 都写求职信吗？

A：建议对重点岗位（B+ 或 官网投递）写定制求职信，批量投递可使用精简版。

### Q：没有具体 JD 怎么办？

A：使用通用版求职信，强调个人核心优势和职业方向。

### Q：可以复用求职信模板吗？

A：可以复用结构，但必须定制具体案例和数据，禁止套用模板原文。

### Q：求职信长度多少合适？

A：中文 300-500 字，英文 250-400 words。精简版 100-200 字。

## 与简历 Skill 联动

当用户请求生成求职信时：

1. **确认授权**：询问是否调用简历 Skill
2. **获取简历**：从简历 Skill 输出中获取优化后的简历
3. **提取 JD**：获取目标岗位的 JD 内容
4. **生成求职信**：按本 Prompt 生成定制求职信
5. **保存文件**：保存到 `~/.getjobs/output/cover_letters/`
6. **更新追踪**：在投递记录中标记"已生成求职信"
