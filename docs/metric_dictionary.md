# 指标词典

指标用于帮助追问和选择证据，不得直接填入用户简历。每个指标必须确认口径、周期、基线、来源、个人贡献和是否可公开。

## 各领域常用指标

### 产品经理

| 指标 | 说明 | 追问重点 |
|------|------|----------|
| DAU/MAU | 日活/月活用户数 | 用户口径、时间窗口、版本变化 |
| 留存率 | 次日/7日/30日留存 | 定义口径、影响因素 |
| 转化率 | 页面到行动的比率 | 漏斗位置、基准对比 |
| 点击率（CTR） | 点击/曝光 | 广告位置、内容类型 |
| 付费率 | 付费用户/总用户 | 付费点设计、用户分层 |
| NPS | 净推荐值 | 问卷口径、样本量 |
| 需求上线数 | 迭代交付量 | 统计周期、需求定义 |

### 运营

| 指标 | 说明 | 追问重点 |
|------|------|----------|
| 拉新数 | 新增用户数 | 渠道、质量、统计口径 |
| 活跃率 | 活跃用户/总用户 | 活跃定义、时间窗口 |
| 复购率 | 重复购买/总购买 | 周期、复购定义 |
| 社群人数 | 社群总人数 | 社群类型、活跃度 |
| 活动参与率 | 参与/曝光 | 活动类型、统计口径 |
| GMV | 商品交易总额 | 含/不含退款、统计口径 |
| ROI | 投入产出比 | 投入口径、产出归因 |

### 销售

| 指标 | 说明 | 追问重点 |
|------|------|----------|
| 线索数（Leads） | 潜在客户数 | 来源、定义、质量 |
| 商机数（Opportunities） | 有效意向客户 | 确认标准、转化口径 |
| 成交率 | 成交/商机 | 周期、个人配额 |
| 回款额 | 已回款金额 | 回款周期、合同口径 |
| 客单价 | 成交金额/客户数 | 客户类型、产品线 |
| 销售周期 | 线索到成交时长 | 周期定义、瓶颈阶段 |
| 续约率 | 续约/总合同 | 客户分层、续约口径 |

### 技术

| 指标 | 说明 | 追问重点 |
|------|------|----------|
| QPS | 每秒查询数 | 峰值/均值、测试场景 |
| 响应时间 | 接口响应时长 | P50/P95/P99、慢查询比例 |
| 可用性 | 正常运行时间比例 | 统计周期、事故影响 |
| 故障率 | 故障次数/总时长 | 故障定义、影响范围 |
| 代码覆盖率 | 被测代码/总代码 | 覆盖率类型、统计口径 |
| 性能提升 | 优化前后对比 | 测试环境、基准确定 |

### 数据分析

| 指标 | 说明 | 追问重点 |
|------|------|----------|
| 看板数量 | 搭建的看板数 | 使用者、频率、更新方式 |
| 指标体系 | 定义的指标数 | 业务覆盖、数据来源 |
| SQL 查询效率 | 查询耗时 | 数据量、查询复杂度 |
| 实验显著性 | A/B 测试结果 | 样本量、置信度、周期 |
| 建议采纳率 | 被采纳建议/总建议 | 采纳标准、业务影响 |

### 市场/增长

| 指标 | 说明 | 追问重点 |
|------|------|----------|
| 曝光量 | 内容展示次数 | 渠道、去重口径 |
| 点击率（CTR） | 点击/曝光 | 渠道、内容类型 |
| CPL | 单条线索成本 | 渠道、线索定义 |
| CPA | 单个获客成本 | 渠道、转化口径 |
| 转化率（CVR） | 转化/曝光 | 漏斗位置、基准 |
| LTV | 用户生命周期价值 | 计算周期、模型假设 |

### 设计

| 指标 | 说明 | 追问重点 |
|------|------|----------|
| 设计交付效率 | 需求交付速度 | 统计周期、需求类型 |
| 组件复用率 | 复用组件/总组件 | 复用定义、评估方式 |
| 用户满意度 | 用户评分/反馈 | 评估方式、样本量 |
| 转化率提升 | 优化前后对比 | 测试场景、周期 |

## 没有精确数据时怎么办

### 可使用的定性描述

| 类型 | 示例 | 使用场景 |
|------|------|----------|
| 规模描述 | "覆盖 X 个城市" | 无法给出精确数据 |
| 频次描述 | "每周更新" | 时间维度 |
| 范围描述 | "A/B/C 三类用户" | 用户分层 |
| 趋势描述 | "提升约 20%" | 有方向但无精确值 |
| 比较描述 | "比之前减少 3 步" | 有改善但无精确值 |

### 可使用的可验证结果

| 类型 | 示例 | 可验证方式 |
|------|------|------------|
| 按时上线 | "按期上线" | 验收记录 |
| 通过验收 | "通过产品验收" | 验收签字/记录 |
| 被业务采用 | "建议被采用" | 邮件/聊天记录 |
| 减少操作 | "减少 3 个步骤" | 操作前后对比 |
| 解决风险 | "避免线上故障" | 事故记录 |

### 禁止行为

- ❌ 制造无来源的精确数字
- ❌ 估算后声称精确
- ❌ 用"约"来编造数字
- ❌ 把可能当必然

## 数据分析深度等级

| 等级 | 可证明的能力 | 不能声称的能力 |
|------|--------------|------------------|
| D0 无证据 | 无 | 数据分析能力 |
| D1 查看报表 | 查看后台、整理报表、基础取数 | 漏斗诊断、业务洞察、决策支持 |
| D2 诊断分析 | 定义口径、分群、趋势/漏斗分析并发现问题 | 建议已推动结果、实验归因 |
| D3 决策支持 | 分析建议被采纳，并有可验证后续变化 | 严格因果、实验体系 |
| D4 实验/体系 | 实验设计、归因、预测或指标体系，方法边界清楚 | 超出实际方法深度的能力 |

**重要**：技能名称不等于分析深度。只写 SQL、Excel、BI，不能自动判定为 D2 以上。

## 成果证据等级

| 等级 | 说明 | 示例 | 使用规则 |
|------|------|------|----------|
| R0 无结果 | 无可展示成果 | "负责活动运营" | 必须追问或仅保留动作 |
| R1 可验证交付 | 有交付物 | 按期上线、完成验收、交付给业务使用 | 可作为结果，但不能写成增长 |
| R2 可验证采用/改善 | 有改善证据 | 建议被采纳、步骤减少、减少人工操作、解决关键风险 | 说明验证方式和影响范围 |
| R3 量化结果 | 有数据支撑 | 转化率变化、耗时下降、工单减少，且有周期/来源/归因边界 | 可量化表达，保留估算和归因限制 |

**重要**：没有数字不等于没有成果；但定性结果必须可由交付物、采用记录、流程变化、反馈或验收证明。

## 贡献动词使用规则

| 动词 | 最低事实门槛 | 使用场景 |
|------|---------------|----------|
| 支持/协助 | 完成明确分配任务，缺少模块所有权 | 辅助性工作 |
| 参与 | 在项目中承担可说明的部分动作，但不负责整体结果 | 部分贡献 |
| 负责 | 对明确模块或交付物有所有权，并能说明从开始到交付的动作 | 主要贡献但非主导 |
| 推动 | 主动协调关键相关方解决阻碍，使明确事项向前推进 | 协调主导 |
| 主导 | 对目标、方案/关键决策、推进节奏和结果责任承担主要责任 | 完全主导 |

**规则**：动词选择以最低已确认证据为准；存在争议时使用较弱动词。
