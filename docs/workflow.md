# 工作流程详解

本文档详细说明 GetJobs 的完整工作流程。

## 整体流程图

```
输入 → 路由判定 → 执行阶段 → 输出
```

---

## 阶段响应路由

### 路由判定逻辑

```
首先判定用户输入属于哪种场景：

1. 有简历 + 有 JD → 路由 A 或 A2
2. 只有简历 → 路由 B
3. 只有 JD → 路由 C
4. 信息严重不足 → 路由 D
5. 造假请求 → 路由 E
```

### 路由 A：简历 + JD 充分

**判定条件**：同时有可解析简历和明确 JD

**执行流程**：完整 8 步流程

**必须输出**：
- 完整报告
- 定制简历
- ATS 版
- HR 摘要
- 面试追问
- 风险提示

**禁止输出**：未确认事实

### 路由 A2：简历 + 多个 JD

**判定条件**：有可解析简历和两个以上明确 JD

**执行流程**：
1. 先建立唯一事实底稿
2. 为每个 JD 独立执行 Step 2-8
3. 输出版本差异表

**必须输出**：
- 每个 JD 的独立匹配表
- 版本差异表
- 各版本简历与风险

**禁止输出**：
- 万能混合版
- 跨版本改变事实

### 路由 B：仅简历

**判定条件**：有简历，无明确 JD

**执行流程**：Step 1 → Step 3 → 部分 Step 5/7

**必须输出**：
- 简历诊断
- 基础 ATS 结构检查
- 通用面试风险分析
- 素材追问问题

**禁止输出**：
- JD 匹配分
- JD 定制版
- 关键词覆盖结论

### 路由 C：仅 JD

**判定条件**：有 JD，无简历

**执行流程**：Step 1 → Step 2 → 部分 Step 5

**必须输出**：
- JD 要求表
- 淘汰风险
- 简历素材清单

**禁止输出**：
- 用户匹配结论
- 任何简历版本
- 五项总分

### 路由 D：信息严重不足

**判定条件**：无可解析简历且无明确 JD

**执行流程**：仅 Step 1

**必须输出**：
- 缺失项清单
- 最多 6 个具体问题
- 明确阻断说明

**禁止输出**：评分、简历重写、ATS 版、匹配结论

### 路由 E：造假请求

**判定条件**：用户要求编造或篡改事实

**执行流程**：立即停止后续步骤

**必须输出**：
- 明确拒绝
- 风险说明
- 可使用的真实表达
- 待补充证据问题

**禁止输出**：评分、虚假重写、ATS 版、最终报告

## 8 步工作流详解

### Step 1：收集输入

**目标**：确认材料、目标、转行状态与隐私边界

**执行动作**：
1. 确认简历内容
2. 确认 JD 内容（用户提供或抓取）
3. 确认求职目标（转行/应届/社招）
4. 确认背景信息
5. 提醒脱敏

**脱敏提醒**：
- 电话、身份证、住址
- 客户机密
- 未公开数据

**输出**：
```markdown
## 输入确认

| 材料 | 状态 | 说明 |
|------|------|------|
| 简历 | ✅ 已提供 | 2024届 |
| JD | ✅ 已提供 | 产品经理 |
| 求职目标 | ✅ 已明确 | 社招转行 |
| 转行状态 | ⚠️ 需确认 | - |
```

---

### Step 2：解析 JD

**目标**：把 JD 解析成可用的要求清单

**执行动作**：
1. 提取岗位名称
2. 提取硬性条件（学历、年限、证书）
3. 提取核心技能
4. 提取核心经验
5. 提取加分项
6. 识别淘汰风险
7. 设置权重

**JD 来源优先级**：
1. 官网招聘页（最准确）
2. 用户直接提供
3. API 抓取
4. 浏览器自动化

**输出**：
```markdown
## JD 要求表

| 要求 | 类别 | 权重 | 原文依据 | 风险 |
|------|------|------|----------|------|
| 5年+经验 | 硬性 | 高 | "5年以上" | 可能被机筛 |
| 本科+ | 硬性 | 高 | "本科及以上" | - |
| SQL | 核心技能 | 高 | "熟练使用SQL" | - |
```

---

### Step 3：解析简历

**目标**：把简历转换为可验证的事实底稿

**执行动作**：
1. 提取教育背景
2. 提取工作经历
3. 提取项目经历
4. 提取技能清单
5. 标记证据状态
6. 标记数据深度
7. 标记成果等级
8. 标记贡献动词上限

**数据深度等级**：
- D0：无证据
- D1：数据查看/报表
- D2：诊断分析
- D3：决策支持
- D4：实验/体系

**成果等级**：
- R0：无结果
- R1：可验证交付
- R2：可验证采用/改善
- R3：量化结果

**输出**：
```markdown
## 简历事实底稿

| 经历 | 原文 | 已知事实 | 数据深度 | 成果等级 | 动词上限 |
|------|------|----------|----------|----------|----------|
| 订单系统 | "负责需求收集" | 负责PRD和原型 | D1 | R2 | 负责 |
```

---

### Step 4：缺口分析

**目标**：逐项映射 JD 与简历证据

**状态定义**：
- ✅ 已覆盖：直接证据
- ⚠️ 弱覆盖：关联但缺深度
- ❌ 未覆盖：无证据
- ❓ 可补充：需追问
- 🚫 不建议硬凑：缺必要资历

**执行动作**：
1. 逐项对照 JD 要求
2. 标记证据状态
3. 计算匹配分
4. 识别淘汰风险

**输出**：
```markdown
## 缺口分析表

| JD 要求 | 权重 | 简历证据 | 状态 | 判断依据 |
|---------|------|----------|------|----------|
| 5年+经验 | 高 | 3年+ | ⚠️ 弱覆盖 | 年限略低 |
| SQL | 高 | "偶尔取数" | ⚠️ 弱覆盖 | 无分析深度 |
```

---

### Step 5：素材追问

**目标**：追问会改变简历结论的关键缺口

**规则**：
- 每轮最多 8 个问题
- 问题要具体
- 说明对应哪个 JD 要求
- 不暗示答案

**追问方向**：
1. 背景和目标
2. 个人负责范围
3. 具体动作
4. 方法/工具
5. 协作对象
6. 结果和数据
7. 复盘

**输出**：
```markdown
## 需要补充的信息

1. **SQL 使用场景**（对应 JD "熟练使用SQL"）
   请描述你用 SQL 具体做过什么？查询频率？

2. **跨团队协作**（对应 JD "跨团队推进能力"）
   请描述你协调过哪些人？具体推进了什么？
```

---

### Step 6：简历重写

**前置门槛**：所有高权重追问已回答

**重写原则**：
- 只用已确认事实
- 按 JD 相关度排序
- 使用真实贡献动词
- 删除低相关重复

**贡献动词门槛**：
- 支持/协助：完成分配任务
- 参与：承担部分动作
- 负责：拥有模块所有权
- 推动：协调解决阻碍
- 主导：承担主要责任

**输出**：
```markdown
## 定制简历

### JD 定制版
[完整简历内容]

### ATS 纯文本版
[可直接复制的纯文本版本]

### HR 快速摘要
[100字内概括]

### Boss 直聘开场白
[50字内]
```

---

### Step 7：风险检查

**检查维度**：
1. ATS 检查
2. HR 视角检查
3. 面试追问生成
4. 真实性审查

**高风险项必须阻断**：
- 编造事实
- 夸大贡献
- 无证据关键词

**输出**：
```markdown
## 风险提示

### 高风险
- 年限略低于 JD 要求，可能被机筛

### 中风险
- SQL 描述需更具体

### 面试追问
| 内容 | 可能追问 |
|------|----------|
| 负责订单系统 | 具体负责哪个模块？ |
```

---

### Step 8：最终报告

**前置条件**：所有高风险项已解决

**报告结构**：
1. 简历诊断结论
2. 五项评分
3. 关键优势与缺口
4. 待补充问题
5. 优化后简历
6. ATS 版本
7. Boss/猎头话术
8. 面试追问
9. 风险提示
10. 下一步建议

## 投递追踪流程

### 状态流转

```
pending ──▶ viewed ──▶ interview ──▶ offer
                │
                └──▶ rejected
pending ──▶ rejected
pending ──▶ withdrawn
```

### 追踪操作

| 操作 | 说明 |
|------|------|
| 创建 | 添加新投递记录 |
| 更新 | 修改状态、备注 |
| 查询 | 按条件筛选 |
| 统计 | 周报/月报 |
| 导出 | CSV/Markdown |

## 求职信生成流程

### 触发条件

- 用户明确请求生成求职信
- JD 和简历已确认

### 生成规则

- 内容与简历一致
- 具体数据/案例
- 针对公司和岗位
- 无模板套话
- 长度适当（300-500字）

### 输出格式

- 标准版（中文 300-500字）
- 精简版（100-200字，用于 BOSS 直聘）
- 英文版（250-400 words）
