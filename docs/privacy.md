# 隐私保护政策

本文档说明 GetJobs Skill 在数据处理过程中的隐私保护原则和实践。

## 核心原则

### 1. 本地优先

- **所有数据存储在本地**：投递记录、简历、JD 缓存均保存在 `~/.getjobs/` 目录
- **不上传云端**：除非用户明确授权，否则不将任何数据上传到第三方服务
- **可离线运行**：核心功能不依赖网络连接

### 2. 最小数据收集

- **只收集必要数据**：仅收集求职相关数据，不收集与求职无关的信息
- **不保存敏感凭证**：不保存任何登录凭证、密码、API 密钥等
- **按需脱敏**：对敏感信息按需脱敏处理

### 3. 用户控制权

- **完全透明**：用户可随时查看、导出、删除自己的数据
- **自主决策**：所有关键操作需用户明确确认
- **可导出**：支持将数据导出为标准格式

### 4. 安全性

- **文件权限**：敏感文件设置适当权限
- **备份机制**：写入前自动备份，防止数据丢失
- **加密存储**（可选）：敏感数据可选择加密存储

## 数据分类

### A 类：必须本地存储

| 数据类型 | 说明 | 存储位置 |
|----------|------|----------|
| 投递记录 | 公司、岗位、状态等 | `~/.getjobs/applications.json` |
| JD 缓存 | 抓取的职位描述 | `~/.getjobs/cache/jds/` |
| 生成简历 | 定制的简历版本 | `~/.getjobs/output/resumes/` |

### B 类：用户控制存储

| 数据类型 | 说明 | 存储位置 | 用户控制 |
|----------|------|----------|----------|
| 用户简历 | 主简历文件 | `~/.getjobs/resume.md` | 用户提供，可选 |
| 求职信 | 生成的求职信 | `~/.getjobs/output/` | 生成前确认 |
| 分析报告 | 匹配分析报告 | `~/.getjobs/output/` | 生成前确认 |

### C 类：不存储

| 数据类型 | 处理方式 |
|----------|----------|
| 登录凭证 | 仅在会话期间使用，不保存 |
| 银行卡信息 | 完全不处理 |
| 身份证号 | 完全不处理 |
| 家庭住址 | 建议脱敏或删除 |
| 真实姓名（可选） | 建议使用昵称 |

## 敏感信息处理

### 必须脱敏的信息

在保存到本地文件前，以下信息**必须**脱敏：

| 信息类型 | 脱敏方式 |
|----------|----------|
| 电话号码 | 删除或替换为 `[已脱敏]` |
| 身份证号 | 完全删除 |
| 银行卡号 | 完全删除 |
| 详细家庭住址 | 删除或保留城市 |
| 紧急联系人 | 删除 |
| 婚姻状况 | 删除 |
| 照片/头像 | 不保存 |

### 建议脱敏的信息

以下信息**建议**脱敏：

| 信息类型 | 建议方式 |
|----------|----------|
| 真实姓名 | 可使用昵称/英文名 |
| 邮箱 | 可使用专用求职邮箱 |
| 社交媒体账号 | 按需保留 |
| 前雇主详细信息 | 可用「某互联网公司」 |

### 客户/公司信息

| 场景 | 处理方式 |
|------|----------|
| JD 中的客户名称 | 可保留或替换为「某客户」 |
| 保密项目名称 | 替换为通用描述 |
| 未公开数据 | 删除或替换为「[数据已脱敏]` |

## 数据使用限制

### 允许的使用

1. **本地分析**：基于用户数据生成匹配报告
2. **文件输出**：生成简历、求职信、报告
3. **追踪更新**：更新投递状态
4. **用户请求**：用户明确要求的数据导出

### 禁止的使用

1. **自动上传**：未经用户同意自动上传数据
2. **第三方共享**：不与任何第三方服务共享数据
3. **商业用途**：不将用户数据用于商业分析
4. **公开披露**：不公开任何用户数据

## 平台交互隐私

### 浏览器抓取

- **不保存登录凭证**：不记录任何平台的登录信息
- **不绕过认证**：不尝试抓取需要登录的内容
- **遵守 robots.txt**：尊重平台的爬虫规则
- **限制频率**：避免短时间内大量请求

### API 调用

- **不保存 API 密钥**：密钥仅在会话期间使用
- **最小权限**：只请求必要的数据访问权限
- **加密传输**：使用 HTTPS 进行通信

## 用户权利

### 查看权

用户可随时查看所有存储的数据：

```markdown
查看我的投递记录
查看我的 JD 缓存
查看生成的简历
```

### 导出权

用户可随时导出数据：

```markdown
导出所有数据为 JSON
导出投递记录为 CSV
导出投递记录为 Markdown
```

### 删除权

用户可随时删除数据：

```markdown
删除某条投递记录
清空 JD 缓存
删除所有数据
```

### 拒绝权

用户可拒绝任何数据收集：

```markdown
不保存此 JD
不使用缓存
不生成分析报告
```

## 数据保留

### 保留期限

| 数据类型 | 保留期限 | 说明 |
|----------|----------|------|
| 投递记录 | 用户删除前 | 长期保留 |
| JD 缓存 | 30 天 | 自动清理旧缓存 |
| 分析报告 | 用户删除前 | 保留至用户删除 |
| 备份文件 | 5 份滚动 | 自动清理旧备份 |

### 自动清理

- JD 缓存超过 30 天自动清理
- 可手动禁用自动清理
- 清理前提示用户确认

## 安全建议

### 用户侧

1. **文件权限**：确保 `~/.getjobs/` 目录权限为 `700`
2. **定期备份**：定期备份重要数据
3. **敏感信息**：避免在简历中保留过多敏感信息
4. **公共设备**：不在公共设备上使用

### 系统侧

1. **文件加密**：敏感数据可选择加密存储
2. **访问日志**：记录数据访问操作
3. **异常检测**：检测异常的数据访问模式

## 合规说明

### 适用法规

- 《个人信息保护法》（PIPL）
- 《数据安全法》
- GDPR（若涉及欧盟用户）

### 数据跨境

- **不主动跨境传输**：所有数据存储在本地
- **用户主动**：仅在用户明确要求时协助导出

## 隐私声明模板

### 用户首次使用提示

```markdown
## 隐私提示

GetJobs Skill 将把以下数据存储在本地：

- 投递记录（~/.getjobs/applications.json）
- JD 缓存（~/.getjobs/cache/）
- 生成的简历（~/.getjobs/output/）

**不会**存储：
- 登录凭证
- 身份证号
- 银行卡信息

您可以随时查看、导出或删除您的数据。

继续使用即表示您同意此隐私政策。
```

### 数据收集确认

```markdown
## 数据使用确认

即将执行以下操作：

1. 抓取 JD 并缓存到本地
2. 基于简历分析匹配度
3. 生成定制简历和求职信

是否确认继续？（是/否）

如需调整隐私设置，请回复「隐私设置」。
```
